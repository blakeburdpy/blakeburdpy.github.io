SET a TO expression
Evaluates expression and then assigns a copy of the result to the variable a.

DISPLAY(expression)
Block
DISPLAY expression
Displays the value of expression, followed by a space.
Text
INPUT()
Block
INPUT

Arithmetic Operators and Numeric Procedures
Text
a + b
a - b
a * b
a / b
Block
(a + b)
(a - b)
(a * b)
(a / b)
The arithmetic operators +, -, *, and / are used to perform arithmetic on a and b.
For example, 17 / 5 evaluates to 3.4.
The order of operations used in mathematics applies when evaluating expressions.
Text
a MOD b
Block
(a MOD b)
Evaluates to the remainder when a is divided by b. Assume that a is an integer ≥ 0 and b is an integer > 0.
For example, 17 MOD 5 evaluates to 2.
The MOD operator has the same precedence as the * and / operators.
Text
RANDOM(a, b)
Block
RANDOM FROM a TO b
Generates and returns a random integer from a to b, including a and b. Each result is equally likely to occur.
For example, RANDOM(1, 3) could return 1, 2, or 3.
Relational and Boolean Operators
Text
a = b
a ≠ b
a > b
a < b
a ≥ b
a ≤ b
Block
(a = b)
(a ≠ b)
(a > b)
(a < b)
(a ≥ b)
(a ≤ b)
The relational operators =, ≠, >, <, ≥, and ≤ are used to test the relationship between two variables, expressions, or values. A comparison using relational operators evaluates to a Boolean value.
Text
NOT condition
Block
NOT (condition)
Evaluates to true if condition is false; otherwise evaluates to false.
Text
condition1 AND condition2
Block
(condition1) AND (condition2)
Evaluates to true if both condition1 and condition2 are true; otherwise evaluates to false.
Text
condition1 OR condition2
Block
(condition1) OR (condition2)
Evaluates to true if condition1 is true or if condition2 is true or if both are true; otherwise evaluates to false.
Selection

  <block of statements>
}
Block
IF condition
  <block of statements>
The code in block of statements is executed if the Boolean expression condition evaluates to true; no action is taken if condition evaluates to false.
Text

{
  <first block of statements>
}
ELSE
{
  <second block of statements>
}
Block
IF condition
  <first block of statements>
ELSE
  <second block of statements>
The code in first block of statements is executed if condition evaluates to true; otherwise the code in second block of statements is executed.

Procedures and Procedure Calls
Text
PROCEDURE procName(parameter1,
  parameter2, ...)
{
  <block of statements>
}
Block
PROCEDURE procName(parameters)
  <block of statements>
Defines procName as a procedure that takes zero or more arguments. The procedure contains block of statements.
The procedure can be called using procName(arg1, arg2, ...).
Text
PROCEDURE procName(parameter1,
  parameter2, ...)
{
  <block of statements>
  RETURN(expression)
}
Block
PROCEDURE procName(parameters)
  <block of statements>
  RETURN expression
Defines a procedure that returns the value of expression. The RETURN statement may appear at any point and causes an immediate return to the calling statement.
The returned value can be assigned using result ← procName(arg1, arg2, ...).
Text
RETURN(expression)
Block
RETURN expression
Returns the flow of control to the point where the procedure was called and returns the value of expression.
Robot

If the robot attempts to move to a square that is not open or is beyond the edge of the grid, the robot will stay in its current location and the program will terminate.

Text
MOVE_FORWARD()
Block
MOVE FORWARD
The robot moves one square forward in the direction it is facing.
Text
ROTATE_LEFT()
Block
ROTATE LEFT
The robot rotates in place 90 degrees counterclockwise (i.e., makes an in-place left turn).
Text
ROTATE_RIGHT()
Block
ROTATE RIGHT
The robot rotates in place 90 degrees clockwise (i.e., makes an in-place right turn).
Text
CAN_MOVE(direction)
Block
CAN MOVE (direction)
Evaluates to true if there is an open square one square in the direction relative to where the robot is facing; otherwise evaluates to false. The value of direction can be left, right, forward, or backward.
